## 🎯 修订背景

### 8.2 合规检查清单

基于中国网络安全相关法规的内容合规检查。

**检查项**：

#### 8.2.1 政治敏感内容
- [ ] 不包含违反国家法律法规的内容
- [ ] 不包含损害国家荣誉的内容
- [ ] 不包含煽动民族仇恨的内容

#### 8.2.2 违法违规内容
- [ ] 不包含色情、暴力、恐怖主义内容
- [ ] 不包含赌博、诈骗信息
- [ ] 不包含邪教、迷信内容

#### 8.2.3 网络安全
- [ ] 不传播病毒、木马等恶意代码
- [ ] 不提供黑客工具、教程
- [ ] 不传播虚假信息、谣言

#### 8.2.4 知识产权
- [ ] 不侵犯版权、商标权
- [ ] 不盗版软件、影视资源
- [ ] 引用内容标注来源

#### 8.2.5 个人信息保护
- [ ] 不泄露他人身份证号
- [ ] 不泄露他人手机号、邮箱
- [ ] 不公开他人家庭住址
- [ ] 不未经授权公开他人照片

#### 8.2.6 数据真实性保障（新增）
基于 2026-02-10 招聘简历评估事件的经验教训：

- [ ] **数据源验证**：确保评估/处理的数据来自正确来源
- [ ] **模拟数据标注**：测试/模拟数据必须有明确警告标注
- [ ] **数据类型标识**：明确标记数据为"真实"/"模拟"/"测试"
- [ ] **来源追溯**：保留数据来源信息（邮件ID、文件路径等）

**模拟数据标注标准**：
```
⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️
⚠️ 警告：本文件为模拟数据，仅供系统测试使用 ⚠️
⚠️ 所有信息均为虚构，请勿作为真实决策依据 ⚠️
⚠️ 真实数据需从原始来源中提取验证 ⚠️
⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️⚠️
```

**数据真实性优先级原则**：
```
数据真实性 > 工作效率 > 输出美观
```

**评分标准**：

### 8.3 发布规范

#### 8.3.1 免责声明
**强制要求**：所有公开发布内容必须附加免责声明。

**模板**：
```markdown
**⚠️ 免责声明**

本文为技术分享和实践经验总结，文中提及的软件、工具和操作方法仅供参考。读者在实际使用过程中应遵守相关软件许可协议，注意数据安全和隐私保护。作者和 OpenClaw AI 助手不对因使用文中所述方法而导致的任何直接或间接损失承担责任。
```

**检查项**：
- [ ] 公开发布内容包含免责声明
- [ ] 免责声明位于文章末尾
- [ ] 分隔符与正文内容隔离

#### 8.3.2 Token 使用统计
**建议要求**：技术分享类内容建议披露 Token 使用情况。

**模板**：
```markdown
## 📊 Token 使用统计

| 指标 | 数值 |
|------|------|
| 输入 Token | ~XX,XXX |
| 输出 Token | ~XX,XXX |
| 总计 | ~XX,XXX |
| 预估成本 | ~¥X.XX |

*统计时间：YYYY-MM-DD*
```

**检查项**：
- [ ] 记录 Token 消耗数据
- [ ] 披露预估成本
- [ ] 标注统计时间

#### 8.3.3 发布顺序规范

**文章结构标准**：
```
1. 正文内容
2. 分隔符 (---)
3. Token 使用统计（如有）
4. 分隔符 (---)
5. 免责声明（必须，最后）
```

### 5.2 配置变更保护（新增）

基于 2026-02-21 配置安全回滚系统的经验，增加以下检查项：

**检查项**：

#### 5.2.1 配置修改前验证
- [ ] **JSON 有效性验证**：保存配置前自动验证 JSON 语法正确性
- [ ] **预备份机制**：修改配置前自动保存当前良好配置
- [ ] **版本标记**：记录每次配置变更的时间戳和版本

#### 5.2.2 配置变更保护机制
- [ ] **Watchdog 守护进程**：支持配置修改后启动守护进程监控

- [ ] **自动回滚**：检测到服务无响应时自动回滚至良好配置
- [ ] **失败配置保留**：自动保存损坏配置用于事后分析

| 合格 | 定期备份 + 基本验证 | 50% |
| 不合格 | 无保护机制 | 0% |

- [ ] **状态查询**：支持查看当前 watchdog 状态
- [ ] **手动回滚**：支持强制触发回滚操作

- [ ] **多重备份**：保留最近多个版本的良好配置

oc-safe-restart -c

#### 3.6 数据完整性保护（新增）

**工作文件处理规范**：

| 操作类型 | 允许/禁止 | 说明 |
|----------|-----------|------|
| 添加标注和说明 | ✅ 允许 | 在不改变原数据的前提下添加警告 |
| 整理格式 | ✅ 允许 | 优化展示格式，不改变内容 |
| 提取摘要 | ✅ 允许 | 创建摘要副本 |
| 创建副本加工 | ✅ 允许 | 在副本上进行处理，保留原始文件 |
| **修改原始数据内容** | **❌ 严禁** | 包括姓名、金额、日期等 |
| **模拟数据冒充真实数据** | **❌ 严禁** | 必须明确区分标注 |
| **未经标注分享测试数据** | **❌ 严禁** | 分享前必须添加标注 |
| **删除/掩盖数据来源** | **❌ 严禁** | 必须保留数据来源信息 |

### 维度 4：监控与日志（补充内容）

#### 4.4 事件复盘机制（新增）

**数据安全事件响应流程**：

```
发现问题 → 立即停止 → 原因分析 → 整改实施 → 复盘记录 → 预防措施
```

**检查项**：
- [ ] **事件记录**：详细记录数据安全事件时间线
- [ ] **根因分析**：识别问题根本原因（流程/技术/人为）
- [ ] **整改措施**：制定并执行立即整改方案
- [ ] **复盘文档**：生成事件复盘报告
- [ ] **预防机制**：建立长期预防措施

**定期审计机制**：
| 审计类型 | 频率 | 内容 |
|----------|------|------|
| 敏感数据文件检查 | 每月 | 检查标注完整性 |
| 数据处理流程审计 | 每季度 | 审计流程执行情况 |
| 事件复盘 | 发生时 | 详细复盘和整改 |

**责任分工**：
| 角色 | 责任 |
|------|------|
| 数据录入 | 确保数据来源正确，模拟数据必须标注 |
| 数据处理 | 不改变原始数据，添加标注而非修改内容 |
| 数据审核 | 验证数据真实性，检查标注完整性 |
| 系统维护 | 定期更新数据标注规范，优化提取流程 |

### 6.3 搜索服务可靠性（新增）

基于 2026-02-22 Brave Search 死循环问题，增加搜索服务可靠性检查：

**检查项**：

#### 6.3.1 搜索 API 可靠性
- [ ] **可控重试**：搜索失败时单次调用，失败即返回（非无限重试）
- [ ] **优雅降级**：搜索失败时系统仍可正常运行
- [ ] **资源隔离**：搜索请求不阻塞 Gateway 主服务

#### 6.3.2 搜索服务选型
- [ ] **API 稳定性**：选择稳定的搜索 API 服务
- [ ] **AI 友好**：支持结构化 JSON 输出，内置 AI 摘要
- [ ] **响应速度**：平均响应时间 < 2 秒

**推荐方案 - Tavily API**：
```bash
# 申请 API Key: https://tavily.com/

# 配置环境变量
export TAVILY_API_KEY="tvly-xxx"

# 优势对比
| 特性 | Brave Search | Tavily |
|------|--------------|--------|
| 失败处理 | 无限重试 | 单次调用 |
| 输出格式 | HTML 需解析 | JSON 直接可用 |
| AI 摘要 | 无 | 内置 answer 字段 |
| 响应时间 | 2-5s | 1-2s |
```

| 等级 | 描述 | 分数 |
